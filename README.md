# Customer Shopping Behaviour Data Analyst Project

## Overview

This project demonstrates an end-to-end Data Analytics workflow using Python, PostgreSQL, and Power BI. The objective is to transform raw retail transaction data into meaningful business insights through data cleaning, exploratory data analysis (EDA), SQL analysis, and interactive dashboards.

The project covers the complete analytics lifecycle, from loading raw data to presenting insights in a professional report and presentation.

---

## Dataset

The dataset contains retail transaction records with the following information:

- Transaction ID
- Customer ID
- Product Category
- Quantity
- Price
- Payment Method
- Store Location
- Transaction Date

The data is analyzed to identify sales trends, customer purchasing behavior, and overall business performance.

## Project Workflow

### 1. Data Loading

- Imported the dataset using Pandas.
- Checked data types and dataset structure.
- Identified missing values and duplicate records.

### 2. Data Cleaning

- Removed duplicate records.
- Converted date columns to the correct format.
- Corrected data types.
- Validated the cleaned dataset.

### 3. Exploratory Data Analysis (EDA)

Performed data analysis to understand:

- Sales distribution
- Product category performance
- Payment method usage
- Store-wise sales
- Monthly sales trends
- Customer purchasing patterns

### 4. PostgreSQL Analysis

Loaded the cleaned dataset into PostgreSQL and performed SQL analysis using:

- Aggregate Functions
- GROUP BY
- ORDER BY
- HAVING
- CASE Statements
- Joins
- Date Functions
- Business KPI Queries

### 5. Power BI Dashboard

Designed an interactive dashboard containing:

- Total Sales
- Total Transactions
- Average Order Value
- Sales by Category
- Sales by Store Location
- Monthly Sales Trend
- Payment Method Distribution
- Interactive Filters

### 6. Reporting

Prepared:

- Business Insights Report
- Power BI Dashboard
- Project Presentation using Gamma
